# 剪贴板后台监听

当你复制内容时，章鱼速贴会自动捕捉为「剪贴」，并写入「剪贴历史」。  
目标：**复制即记录，形成可搜索、可同步、可复用的数据流。**

## 你可以做什么

- 自动记录每一次复制（文本、图片、文件/路径）
- 随时在「剪贴历史」中检索、复用或删除
- 触发后续能力（附近同步、云同步、快速输入等）

## 30 秒上手

1. **macOS / Windows**：打开应用 → 自动后台监听  
2. **Android**：任选一种授权方式（**Shizuku 推荐**）→ 开启后台监听  
3. **iOS**：用「系统分享」或「快捷指令」把复制内容发送到章鱼速贴

## 平台与差异

| 平台 | 是否支持自动后台监听 | 如何启用 | 是否需要权限 | 限制/说明 |
|------|----------------------|----------|--------------|-----------|
| iOS | 否 | 系统分享 / 快捷指令 / 打开 App 触发读取 | 否 | 系统不允许持续后台读取（规划支持剪贴键盘、PiP 临时后台） |
| Android | 是 | Shizuku / 无障碍 / Root（三选一） | 视模式而定 | Android 10 以下无需配置；10+ 需授权 |
| macOS | 是 | 打开应用即启用 | 否 | 系统级后台运行 |
| Windows | 是 | 打开应用即启用 | 否 | 系统级后台运行 |
| Linux（规划） | 部分 | 托盘常驻 | 待定 | 预计先适配 X11，Wayland 后续适配 |

## 前置条件

- **Android 10+**：需要授权其一 → Shizuku / 无障碍 / Root  
- **网络**：与同步类功能无直接依赖；但你可能会在之后开启附近同步/云同步  
- **系统提示**：iOS 14+ 在前台读取时可能显示“已粘贴自…”提示（系统行为）

## 预期与限制

- ✅ 启用后，你复制的内容会**自动**成为「剪贴」，写入「剪贴历史」  
- ✅ 支持文本、图片、文件（含路径），具体可在设置中按剪贴类型过滤  
- ⚠️ **iOS** 无法持续后台监听；需通过系统分享/快捷指令/打开 App 触发  
- ⚠️ **Android** 授权可能被系统回收（省电策略/重启）；需按子页步骤确认保活设置  
- ⚠️ **隐私与提示**：系统层面的剪贴板访问提示由各平台控制，无法屏蔽

## 常见问题

- **Q：我用 iPhone，能自动后台吗？**  
  A：目前不行（系统限制）。你可用「系统分享」或「快捷指令」实现接近自动的流程；后续将提供剪贴键盘与 PiP 临时后台模式。
- **Q：Android 该选哪种方式？**  
  A：优先推荐 **Shizuku**（无需 Root，稳定性好）；不想连电脑可选「无障碍模式」；已 Root 可直接使用 Root 模式。
- **Q：会很耗电吗？**  
  A：桌面端几乎无感；Android 建议按子页中的保活建议设置；iOS 仅在前台/触发时读取。
- **Q：我复制了图片/文件也会记录吗？**  
  A：支持。可在设置中按剪贴类型启用/禁用记录策略。
