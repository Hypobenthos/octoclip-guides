# 剪贴板后台监控

为了做到「复制即记录」，章鱼速贴需要在后台持续关注系统剪贴板的变化，这就是我们所说的「剪贴板后台监控」。

## 背景监控会做什么？

后台监控只做一件事：

> 当系统剪贴板发生变化时，读取一次新的内容，并把它保存为剪贴。

它不会：

- 不会记录你的键盘输入
- 不会截屏或录屏
- 不会在你没有复制内容时“偷看”任何东西

技术上它就是：**监听「剪贴板变更事件」 → 读取当前剪贴板内容 → 写入剪贴历史**。

## 各平台的运行方式（简要说明）

根据不同平台的限制，后台监控的表现会略有不同：

- **Windows / macOS**：
章鱼速贴在打开后可以在后台常驻进程中，可靠地持续监听剪贴板变更。
- **Android**：
由于系统限制，后台监控可能依赖前台服务、辅助功能或系统提供的特定接口，在部分机型或系统策略下可能会被系统暂停或回收。
- **iOS**：
- 由于系统限制，章鱼速贴只能在应用处于前台时监听剪贴板变更。我们提供了「快捷指令」等辅助工具，帮助你在一定程度上实现「复制即记录」。

你在界面中看到的「来源 -> 剪贴板 -> 启用来源」或者「来源 -> 剪贴板 -> 后台监控」开关，就是用来控制这些行为是否开启。

## 隐私设计原则

章鱼速贴在设计后台监控时遵守几个原则：

- **最小必要性**：只监听剪贴板，不扩展到键盘、屏幕等无关数据
- **本地优先**：所有剪贴优先保存在本地设备上，是否上传或同步由你自行选择
- **可见可控**：你可以在设置中：
- 暂停后台监控
- 清空剪贴历史
- 对特定来源或应用设置“不过滤 / 不记录”

## 什么时候适合关闭后台监控？

如果你：

- 正在处理高度敏感的信息（证件号、密码等），且不希望被记录
- 正在使用公共或临时设备
- 仅短期使用章鱼速贴的某项功能，而不需要长时间记录剪贴

你可以暂时关闭后台监控，或开启应用排除列表，只对特定场景不记录剪贴。
