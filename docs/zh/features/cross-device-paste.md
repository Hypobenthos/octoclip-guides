# 跨设备粘贴

「跨设备粘贴」是整个跨设备能力的最终体验形态：  

> 在 A 设备上复制，在 B 设备上粘贴，**感觉就像在同一台电脑上操作一样自然**。

它本身不是一个单独的开关，而是基于：

- 剪贴历史：负责记录你复制过的内容  
- 附近同步 / 云同步：负责把剪贴送到其他设备  
- 快速输入窗口 / 系统剪贴板写入：负责在目标设备上“帮你粘贴”

## 基本使用路径

以「在电脑上复制，在手机上粘贴」为例：

1. 在电脑上复制一段文本 / 图片 / 文件  
2. 章鱼速贴把这条内容记录为剪贴，并根据你的设置：
   - 通过附近同步发送给同一网络下的手机  
   - 或通过云同步推送到所有在线设备  
3. 在手机上：
   - 打开章鱼速贴的快速输入 / 剪贴历史  
   - 选择这条刚同步过来的剪贴  
   - 一键写入系统剪贴板或直接粘贴到当前输入框

在理想网络环境下，整个过程只需要几秒钟，操作体验接近“本机复制粘贴”。

## 几种典型场景

- **电脑 → 手机**  
  从浏览器复制一段网址，直接在手机的浏览器或聊天窗口里粘贴  
- **手机 → 电脑**  
  在手机上复制验证码 / 一段短文，在电脑端登录页面直接粘贴  
- **工作机 → 私人设备**  
  在公司电脑写了一段说明，下班后在家里的设备继续编辑或发给同事  

## 跨设备粘贴是怎么“串起来”的？

可以用一句话来概括整条链路：

> 复制 → 生成剪贴 → 通过附近同步 / 云同步传输 → 在另一台设备上选择剪贴并粘贴

你只需要关心两点：

1. **剪贴有没有被记录下来？**  
   - 对应功能：剪贴历史 & 剪贴来源  
2. **这条剪贴有没有被送到目标设备？**  
   - 对应功能：附近同步 / 云同步

当这两步都通了，跨设备粘贴就顺理成章了。

## 小提示

- 如果你发现剪贴已经在另一台设备的剪贴历史里，但还没出现在系统剪贴板：  
  - 尝试在该设备上手动点一次「复制到剪贴板」或使用快速输入窗口  
- 如果剪贴历史里都没有：  
  - 检查源设备是否正常记录剪贴（后台监控、排除列表等设置）  
  - 检查两台设备的附近同步 / 云同步状态

我们的目标是做到：

> **复制即记录，记录即可跨设备粘贴。**
